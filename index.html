About CryptVault

Encryption Methods

  • AES-256-GCM ‚Äî Military-grade encryption with built-in authentication. Detects if data was tampered with. Best for passwords & sensitive data.
  • AES-256-CBC ‚Äî Classic block cipher used by banks and governments. Highly reliable and widely compatible.
  • AES-256-CTR ‚Äî Counter mode encryption. Fastest option, great for large texts. Stream-based processing.
  • Triple Layer ‚Äî Maximum security. Encrypts with AES-GCM, then applies additional encoding layers for extra protection.

How Encryption Works Locally

CryptVault uses the Web Crypto API built directly into your browser. Here's what happens:

  • Your password is transformed using PBKDF2 with 100,000 iterations and a random salt
  • A unique 256-bit key is derived for each encryption
  • Random IV (Initialization Vector) ensures identical texts encrypt differently each time
  • Zero network requests ‚Äî everything happens in your browser's secure environment
  • Works completely offline after first load

History Storage

Your encryption history is stored securely:

  • Saved in your browser's localStorage only
  • Only stores previews (first 30 characters) ‚Äî never full encrypted data
  • Maximum 10 recent entries to minimize exposure
  • Never synced to any cloud or server
  • Clearing browser data removes all history
